# Meme Token Analyzer Bot

Telegram-бот, который принимает адрес контракта мем-токена и оценивает его
по 5 критериям, выдавая итоговый скор 0–10.

## Что реально проверяется

| Критерий | Источник | Что оценивается |
|---|---|---|
| Ликвидность | DexScreener | Размер пула ликвидности в USD |
| Объём торгов | DexScreener | Соотношение объёма за 24ч к ликвидности (детект wash trading) |
| Возраст/стадия | DexScreener | Сколько живёт токен + капитализация (тайминг входа) |
| Безопасность контракта | GoPlus Security | honeypot, mint, pause, blacklist, налоги, renounced ownership |
| Держатели | GoPlus Security | % токенов у топ-10 кошельков |

Поддерживаются сети: **Ethereum, BSC, Polygon, Arbitrum, Base, Avalanche,
Optimism, Fantom, Robinhood Chain, Solana, TON** — бот сам определяет сеть
по адресу через DexScreener.

### Про TON и Robinhood Chain отдельно

- **Robinhood Chain** — EVM-совместимый L2 (Arbitrum Orbit, chainId 4663),
  публичный mainnet с 1 июля 2026. DexScreener уже индексирует пары
  (chain slug `robinhood`). GoPlus Security на момент написания бота — это
  очень новая сеть, поддержка контракт-секьюрити под вопросом. Код
  настроен пробовать запрос к GoPlus (chain_id 4663); если сеть не
  поддерживается, соответствующий блок скора корректно уходит в **N/A**,
  бот не падает и не выдумывает данные.
- **TON** — не EVM и не Solana, отдельная архитектура. DexScreener данные
  (ликвидность, объём, возраст, капитализация) работают как обычно
  (chain slug `ton`). GoPlus не имеет официального публичного эндпоинта
  под TON на момент написания — блок "безопасность контракта" и
  "держатели" для TON-токенов будет почти всегда **N/A**. Если появится
  такой эндпоинт (или сторонний вроде tonapi.io с аналогичными
  проверками), это можно добавить в `analyzer.py` в функцию
  `get_goplus_security` отдельной веткой.

## Честные ограничения (важно прочитать)

- **Это НЕ финансовый совет и не гарантия безопасности.** Скор — это
  эвристика на основе публичных ончейн-данных.
- Бот **не оценивает** реальность хайпа в Twitter/Telegram, репутацию
  команды, наличие doxxed-разработчиков — для этого нет надёжного
  бесплатного API.
- Если для какого-то критерия нет данных (например, GoPlus не знает эту
  сеть или контракт), он помечается **N/A** и не участвует в среднем — это
  честнее, чем подставлять произвольное значение.
- GoPlus и DexScreener — бесплатные API с лимитами по частоте запросов.
  При высокой нагрузке бот может получать пустые ответы — это обрабатывается
  gracefully (компонент помечается N/A), но точность может падать.
- Схема JSON-ответов сторонних API может меняться со временем — если
  что-то в разборе полей (`analyzer.py`) сломается, скорее всего дело в
  этом, и нужно свериться с актуальной документацией GoPlus/DexScreener.

## Установка и запуск

1. Получи токен бота у [@BotFather](https://t.me/BotFather) в Telegram
   (команда `/newbot`).

2. Установи зависимости:
   ```bash
   pip install -r requirements.txt
   ```

3. Установи переменную окружения с токеном и запусти:
   ```bash
   export TELEGRAM_BOT_TOKEN="твой_токен_от_BotFather"
   python bot.py
   ```

4. Открой бота в Telegram, нажми `/start`, отправь адрес токена.

## Деплой на бесплатный хостинг (Render)

Из по-настоящему бесплатных вариантов без банковской карты в 2026 году
проще и надёжнее всего — **Render**. Есть нюанс: бесплатный тариф Render
рассчитан на веб-сервисы и "засыпает" после ~15 минут без HTTP-запросов,
а наш бот работает через polling и сам HTTP не принимает. Поэтому:
1. В проект добавлен `keep_alive.py` — лёгкий HTTP-сервер, который
   запускается автоматически, если задана переменная `PORT` (Render
   всегда её прокидывает). Он просто отвечает "OK" на любой запрос.
2. Чтобы сервис не засыпал, нужно настроить внешний "пинг" каждые
   5-10 минут — бесплатно через **UptimeRobot** (шаги ниже).

### Шаг 1. Залей код на GitHub

Если ещё не залито:
```bash
cd memebot
git init
git add .
git commit -m "Meme token analyzer bot"
```
Создай пустой репозиторий на [github.com/new](https://github.com/new),
затем:
```bash
git remote add origin https://github.com/твой-юзернейм/memebot.git
git branch -M main
git push -u origin main
```

### Шаг 2. Создай сервис на Render

1. Зарегистрируйся на [render.com](https://render.com) (можно через
   GitHub-аккаунт, карта не нужна для free tier).
2. **New** → **Web Service** → выбери репозиторий `memebot`.
3. Настройки:
   - **Runtime**: Python 3
   - **Build Command**: `pip install -r requirements.txt`
   - **Start Command**: `python bot.py`
   - **Instance Type**: Free
4. В разделе **Environment** добавь переменную:
   - `TELEGRAM_BOT_TOKEN` = твой токен от BotFather
5. Нажми **Create Web Service**. Через пару минут в логах должно
   появиться `Bot started` и `Keep-alive HTTP server listening on port ...`.
6. Скопируй URL сервиса (вида `https://memebot-xxxx.onrender.com`) —
   он понадобится для следующего шага.

### Шаг 3. Настрой "будильник" через UptimeRobot (бесплатно)

Без этого шага бот будет засыпать через 15 минут простоя и отвечать
с задержкой (первый запрос после сна поднимает сервис ~30-60 сек).

1. Зарегистрируйся на [uptimerobot.com](https://uptimerobot.com) (free tier).
2. **Add New Monitor** → тип **HTTP(s)**.
3. URL: вставь ссылку на свой Render-сервис из шага 2.
4. Interval: 5 минут.
5. Сохрани.

Теперь UptimeRobot будет пинговать бота каждые 5 минут, и Render не
будет усыплять сервис.

### Ограничения такого подхода

- Free tier Render даёт 750 часов/месяц — для одного сервиса 24/7 этого
  достаточно (в месяце ≤744 часа).
- Бесплатные ресурсы CPU/RAM ограничены — для одного бота с умеренной
  нагрузкой хватает, но если бот станет очень популярным, может тормозить.
- Если Render изменит условия free tier (такое уже случалось с другими
  платформами) — понадобится либо перейти на платный план (~$7/мес),
  либо на VPS.

### Альтернатива для более стабильной работы

Если бот наберёт реальную аудиторию, лучше не полагаться на бесплатные
тарифы — они периодически урезаются. Более надёжный вариант — простой
VPS ($4-6/мес, например DigitalOcean/Hetzner) с `systemd`-сервисом,
который перезапускает бота при падении и не имеет ограничений по времени
работы. Могу расписать этот вариант отдельно, если понадобится.

## Хостинг: другие варианты

- **VPS** (например, простой сервер за $4-6/мес) + `systemd`-сервис —
  самый предсказуемый вариант без "засыпаний" и ограничений free tier.
- **Docker**: можно обернуть в контейнер (Dockerfile не включён, но проект
  простой — `FROM python:3.11-slim`, `COPY`, `pip install -r requirements.txt`,
  `CMD ["python", "bot.py"]`).
- **Railway**: в 2026 году у Railway остался только 30-дневный бесплатный
  trial (не постоянный free tier), дальше платно.
- **Fly.io**: с 2026 года требует банковскую карту даже для нового
  бесплатного лимита — раньше был вариант без карты, сейчас нет.

## Возможные доработки

- Кэширование запросов (Redis/in-memory) — снизит нагрузку на API при
  повторных запросах одного адреса.
- Добавить проверку соцсигналов через LunarCrush API (платный) для более
  честной оценки хайпа.
- Хранить историю запросов пользователя (SQLite) для команды `/history`.
